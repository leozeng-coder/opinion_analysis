# 深度分析自适应预算架构 - 实施完成

## 概览

已将深度分析从**固定阈值全量挖掘**重构为**自适应预算分配**架构，解决了以下核心问题：

1. ✅ **评论数/点赞数不可靠** → 改用内容质量+时效+情感强度打分
2. ✅ **固定预算易截断** → 200k总预算，按话题重要性动态分配
3. ✅ **数据量不可控** → 预过滤+轻量打分+分层采样，自动缩放
4. ✅ **低价值数据全量分析** → 高/低价值分离，算力集中到高价值

## 新增文件

### `backend/src/service/report/deep_adaptive.go`

包含自适应预算架构的核心函数：

#### 阶段0: 质量预过滤（零成本）
- `prefilterArticles()` - 文章规则过滤（过短/无内容/陈旧）
- `prefilterComments()` - 评论增强过滤（停用词+纯emoji+复读机）

#### 阶段1: 轻量打分（规则+模糊区LLM）
- `articleScoreRule()` - 规则打分0-10分：内容充分度(4分)+时效(2分)+情感强度(3分)+标签丰富度(1分)
- `lightweightScoreArticles()` - 主流程：全量规则打分 → 识别模糊区(3.5-6.5分) → LLM细化40%数据
- `llmScoreFuzzyBatch()` - 批量LLM判断是否有明确观点（有观点+2分，无观点+0分）

#### 阶段3: 预算分配器
- `topicBudget` 结构 - 单话题预算：深挖文章数、单篇token、评论采样上限
- `budgetAllocator()` - 核心算法：
  - 计算重要性 = 文章数 × 平均分数 × 平均时效权重
  - 按比例分配200k预算，单话题上限50k
  - 超限时线性缩放，保证每话题≥2篇、单篇≥800 tokens

#### 阶段5: 评论自适应采样
- `deepCommentAnalysisAdaptive()` - 按budgets分配的commentSample上限采样（10-100条）
- `analyzeClusterCommentsAdaptive()` - 自适应token预算（800+样本数×25，上限5000）

## 修改文件

### `backend/src/service/report/service.go`

**修改行数**: 240-334行（deepMode分支）

**替换逻辑**:
```
旧: 粗过滤(评论) → 细过滤(文章) → 全量深挖 → 打分 → 聚类 → 评论分析
新: 预过滤(文章+评论) → 轻量打分 → 聚类 → 预算分配 → (简化)高价值过滤 → 评论自适应采样
```

**关键参数**:
- 全局预算上限：200k tokens
- 单话题预算上限：50k tokens
- 超限兜底：线性缩放，保证每话题≥2篇、单篇≥800 tokens

**注意**: 当前实现为**过渡版**，阶段4的分层抽样深挖（`stratifiedSampleInsights` + `deepAnalyzeBatchAdaptive`）暂未完全实现，使用了简化逻辑：
- 直接用轻量打分结果作为DecisionValue
- 过滤≥3分的洞察
- 重新聚类高价值洞察

完整实现需要进一步重构 `deep_analysis.go` 的 `deepAnalyzeAll` 函数。
