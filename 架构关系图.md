## 🔄 数据流转

### 1. 用户触发爬虫

```
用户 → 前端 → Go API (/api/crawler/run-now)
                ↓
         创建 crawler_run_logs 记录
                ↓
         启动子进程: python run_once.py
                ↓
         返回 run_id 给前端
```

### 2. 爬虫执行流程

#### Stage 1: BroadTopicExtraction (broad-topic)

```
run_once.py
    ↓
BroadTopicExtraction.run_daily_extraction()
    ↓
收集 13 个平台新闻 (RSS/API)
    ↓
AI 关键词提取 (DeepSeek)
    ↓
保存到 daily_topics 表
    ↓
ArticleSyncBridge.sync_daily_news_to_articles()
    ↓
同步到 articles 表 (Go 后端使用)
```

#### Stage 2: DeepSentimentCrawling (deep-sentiment)

```
run_once.py
    ↓
读取 daily_topics 表的关键词
    ↓
调用 MediaCrawler (Playwright)
    ↓
爬取各平台数据:
  - 小红书 (xhs_note, xhs_note_comment)
  - 抖音 (douyin_aweme, douyin_aweme_comment)
  - B站 (bilibili_video, bilibili_video_comment)
  - 微博 (weibo_note, weibo_note_comment)
  - 快手 (kuaishou_video, kuaishou_video_comment)
  - 贴吧 (tieba_note, tieba_note_comment)
  - 知乎 (zhihu_question, zhihu_answer)
    ↓
保存到各平台对应的表
```

### 3. 进度更新

```
run_once.py
    ↓
每个阶段更新 crawler_run_logs:
  - progress (0-100)
  - progress_detail (JSON)
  - status (running/success/failed)
    ↓
前端轮询 /api/crawler/progress/:id
    ↓
实时显示进度
```

## 🔗 关键关联点

### 1. Go 后端 → Python 爬虫

**通信方式**: 子进程 + 环境变量

```go
// backend/src/api/handler/user/crawler.go
cmd := exec.CommandContext(ctx, py, script, "--spiders", spiders)
env := append(os.Environ(),
    "DATABASE_DSN="+config.Cfg.Database.DSN,
    "CRAWLER_RUN_LOG_ID="+fmt.Sprintf("%d", logID),
    "CRAWLER_SPIDER_NAMES="+spiders,
    "CRAWLER_FILTER_FILE="+filterFile,
)
cmd.Env = env
```

### 2. Python 爬虫 → 数据库

**连接方式**: 直接 MySQL 连接

```python
# crawler/run_once.py
target_dsn = os.environ.get("DATABASE_DSN")
# 格式: root:123456@tcp(127.0.0.1:3306)/opinion_analysis

bridge = ArticleSyncBridge(target_dsn)
bridge.sync_daily_news_to_articles(date.today())
```

### 3. MediaCrawler → 数据库

**配置文件**: `MediaCrawler/config/db_config.py`

```python
MYSQL_DB_HOST = "127.0.0.1"
MYSQL_DB_PORT = 3306
MYSQL_DB_USER = "root"
MYSQL_DB_PWD = "123456"
MYSQL_DB_NAME = "opinion_analysis"
```

## 📊 数据查询示例

### 查询爬取的文章

```sql
-- 查询最近的文章
SELECT id, title, platform, sentiment, published_at
FROM articles
ORDER BY published_at DESC
LIMIT 10;

-- 按平台统计
SELECT platform, COUNT(*) as count
FROM articles
GROUP BY platform;

-- 查询特定关键词的文章
SELECT title, content, platform
FROM articles
WHERE JSON_CONTAINS(keywords, '"Python"')
ORDER BY published_at DESC;
```

### 查询爬虫运行记录

```sql
-- 查询最近的运行记录
SELECT id, spiders, status, progress, started_at, finished_at
FROM crawler_run_logs
ORDER BY started_at DESC
LIMIT 10;

-- 查询失败的记录
SELECT id, spiders, message
FROM crawler_run_logs
WHERE status = 'failed'
ORDER BY started_at DESC;
```

### 查询 MediaCrawler 原始数据

```sql
-- 小红书笔记
SELECT note_id, title, desc, liked_count
FROM xhs_note
ORDER BY create_time DESC
LIMIT 10;

-- 抖音视频
SELECT aweme_id, desc, author_name, digg_count
FROM douyin_aweme
ORDER BY create_time DESC
LIMIT 10;
```

## 🎯 总结

1. **Go 后端**: 提供 API 接口，管理爬虫调度和数据查询
2. **Python 爬虫**: 执行实际的数据采集工作
3. **MediaCrawler**: 专门负责社交媒体平台的深度爬取
4. **MySQL 数据库**: 统一存储所有数据
5. **数据流转**: Go → Python → MediaCrawler → MySQL → Go → 前端

这种架构的优势：
- ✅ 职责分离：Go 负责业务逻辑，Python 负责爬虫
- ✅ 灵活扩展：可以独立升级爬虫模块
- ✅ 数据统一：所有数据存储在同一个数据库
- ✅ 易于维护：各模块独立开发和测试
